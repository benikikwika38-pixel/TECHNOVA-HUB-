📌 Présentation générale

Firebase est une plateforme Backend-as-a-Service (BaaS) développée par Google. Elle permet de créer rapidement des applications Web et Mobile sans avoir besoin de développer un serveur. Firebase fournit plusieurs services comme :

  • Authentification
  • Cloud Firestore
  • Cloud Storage
  • Firebase Hosting
  • Cloud Functions
  • Analytics

Grâce à Firebase, les développeurs peuvent se concentrer sur leur application pendant que Google gère l'infrastructure.

📖 CHAPITRE 1 : Introduction et prise en main

Concepts clés

  • Projet : conteneur principal regroupant votre application et tous les services Firebase.
  • Console Firebase : interface web permettant de gérer votre projet.
  • SDK : bibliothèque que l'on ajoute dans son projet pour communiquer avec Firebase.
  • Avantage : création rapide d'applications sans gérer de serveur.
  • Limite : moins adapté aux applications très complexes nécessitant une base de données relationnelle.

Étapes de démarrage

  1. Créer un projet sur Firebase.
  2. Ajouter une application Web.
  3. Récupérer la configuration Firebase.
  4. Ajouter le SDK dans votre projet.
  5. Activer les services nécessaires.

📖 CHAPITRE 2 : Authentification des utilisateurs

Firebase Authentication permet de créer facilement un système de connexion sécurisé.

Méthodes de connexion

  • Email / Mot de passe
  • Compte Google
  • Facebook
  • GitHub
  • Numéro de téléphone

Fonctionnalités

  • Création de compte
  • Connexion
  • Déconnexion
  • Réinitialisation du mot de passe
  • Gestion automatique des sessions
const auth = getAuth(app);

Chaque utilisateur possède un identifiant unique appelé UID.

📖 CHAPITRE 3 : Cloud Firestore

Cloud Firestore est une base de données NoSQL fonctionnant en temps réel.

Organisation

  • Collection
  • Document
  • Champ

Chaque document possède un identifiant unique et contient plusieurs champs.

Les opérations CRUD

  • Créer : addDoc() ou setDoc()
  • Lire : getDoc()
  • Modifier : updateDoc()
  • Supprimer : deleteDoc()
Collection ↓ Document ↓ Nom : Béni Âge : 20 Pays : RDC

Firestore permet également d'écouter les modifications en temps réel grâce à onSnapshot().

📖 CHAPITRE 4 : Stockage des fichiers

Firebase propose un service appelé Cloud Storage qui permet de stocker différents types de fichiers.

  • Images
  • Vidéos
  • Documents PDF
  • Fichiers ZIP
  • Audio

Fonctionnalités principales

  • Téléverser un fichier
  • Télécharger un fichier
  • Supprimer un fichier
  • Organiser les fichiers dans des dossiers
  • Partager un lien sécurisé

Bonne pratique : les fichiers sont enregistrés dans Cloud Storage tandis que leurs informations (nom, URL, date...) sont stockées dans Firestore.

Images/ profil.jpg Documents/ certificat.pdf Videos/ cours.mp4

📖 CHAPITRE 5 : Sécurité et règles d'accès

Firebase protège automatiquement vos données grâce aux règles de sécurité.

Par défaut, personne ne peut accéder aux données tant que vous n'avez pas défini des autorisations.

Exemple de règle Firestore

rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /utilisateurs/{uid} { allow read, write: if request.auth != null && request.auth.uid == uid; } } }

Bonnes pratiques

  • Ne jamais laisser la base ouverte à tout le monde.
  • Toujours vérifier si l'utilisateur est connecté.
  • Limiter les droits d'accès selon le rôle de chaque utilisateur.
  • Tester les règles avant de publier l'application.

📖 CHAPITRE 6 : Déploiement et services complémentaires

Lorsque votre application est terminée, Firebase permet de la publier facilement sur Internet grâce à Firebase Hosting.

firebase init hosting firebase deploy

Autres services Firebase

  • Cloud Functions
  • Firebase Hosting
  • Firebase Analytics
  • Crashlytics
  • Cloud Messaging (Notifications)
  • Remote Config
  • Performance Monitoring

Tous ces services fonctionnent ensemble pour créer des applications modernes, rapides et sécurisées.

📚 RÉSUMÉ DU COURS

Firebase est une plateforme Backend-as-a-Service (BaaS) développée par Google. Elle permet de créer rapidement des applications modernes sans gérer un serveur.

Les principaux services sont :

  • Firebase Authentication
  • Cloud Firestore
  • Cloud Storage
  • Firebase Hosting
  • Cloud Functions
  • Analytics

Grâce à Firebase, les données sont synchronisées en temps réel entre tous les appareils connectés.

🎯 À RETENIR

  • ✅ Firebase est développé par Google.
  • ✅ Firestore est une base de données NoSQL.
  • ✅ Cloud Storage permet de stocker les fichiers.
  • ✅ Authentication gère les utilisateurs.
  • ✅ Hosting permet de publier un site Web.
  • ✅ Les règles de sécurité protègent les données.
  • ✅ Firebase fonctionne parfaitement avec HTML, CSS et JavaScript.

📜 HISTOIRE DE FIREBASE

2011 : Firebase est créé par Firebase Inc.

2014 : Google rachète Firebase afin d'en faire une plateforme complète pour les développeurs.

2017 : Google lance Cloud Firestore.

Aujourd'hui, Firebase est l'une des plateformes Cloud les plus utilisées dans le monde pour le développement d'applications Web et Mobile.
